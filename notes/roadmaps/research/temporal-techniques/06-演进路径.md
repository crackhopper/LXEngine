# 06 · 演进路径

> 阅读前提：[05](05-LX当前状态对照.md) 已经盘清 LX 现状与跟前面 5 个调研的耦合。本文给出推进路径切分。
>
> 注：本文给出的 REQ-A..F 编号是 *候选编号*，正式立项时按 `notes/requirements/` 实际可用号顺延。

## 6.1 总体策略

**结论先行：6 个候选 REQ，按"基础设施 → 特性"两层切分**：

- **基础设施层（REQ-A）**：所有 temporal 共享的底盘 — jitter / motion vector / history texture / blue noise，一次到位
- **特性层（REQ-B..F）**：消费基础设施实现具体视觉特性

每个 REQ：

- 有清晰的输入输出
- 完成后工程仍可编译可测
- 可独立合 PR，避免大爆炸式提交
- *仅* REQ-A 是硬前置，其余可按收益排序穿插

**前置条件**：

- frame-graph 候选 REQ-035（FramePass 多 pass）和 REQ-038（barrier 自动推导）必须先就位
- multi-threading/08 的 `IGpuTimeline` + `RetirePoint` 必须落地（history 跨帧 retire）
- gpu-driven-rendering 的 clustered lighting（REQ-F）必须在 REQ-D 之前完成
- shadows 的 cubemap shadow（REQ-F）必须在 REQ-D 之前完成

## 6.2 六步演进序列

### REQ-A · Temporal 共享基础设施

**目标**：建好 temporal 渲染的所有共享底盘，让后续 REQ 即插即用。

**改动**：

- `Camera` 加 jitter API：每帧把 `projection.m20/m21` 偏移 `jitter / resolution`
- `Camera` 缓存 `previous_view_projection`（per-frame 双 buffer）
- 引入 Halton sequence 工具（base 2/3，period 4-16）
- 新增 motion vector buffer 资源类型（R16G16，1080p ≈ 8 MB）
- 实现 camera motion compute pass（从 depth + view-proj reconstruction）
- 扩展 vertex/mesh shader 输出 per-vertex velocity（针对动态物体）
- 引入 history texture 双 buffer 模式（跨帧资源，走 frame-graph 跨帧表达）
- 烘焙 / 加载 blue noise 2D 资产
- 实现 golden ratio 时间偏移工具（`fract(noise + frame * 0.61803398875)`）
- 实现 triangular mapping 工具（两个均匀 → 三角分布）

**前置依赖**：

- frame-graph REQ-035（FramePass 数据结构）— 跨帧资源 + 多 pass 表达
- multi-threading/08 `IGpuTimeline` + `RetirePoint` 落地 — history 跨帧 retire

**验收**：

- 写一个 demo：camera 缓慢平移，可视化 motion vector buffer
- 静态场景 jitter 启用后 sub-pixel sample 分布符合 Halton
- history texture 双 buffer 切换 validation 全绿
- blue noise 采样 + golden ratio 动画化，连续 16 帧无可见偏移

**风险**：中。基础设施铺设面广，但每块都是孤立机制，单点验证。

**可独立合 PR**：是。

### REQ-B · TAA 最简版（baseline）

**目标**：跑通 [04 §4.4](04-TAA.md#44-最简实现理解骨架) 的 4 行骨架版 TAA。

**改动**：

- 新增 TAA compute shader（6 步骨架，但每步用最简实现）
- 在 lighting pass 之后加 TAA pass
- 固定 `0.9 history + 0.1 current` resolve
- 跟 frame graph 集成（TAA pass 消费 history + scene color，输出新 history）
- 跟 sampler 配合：mipLodBias = -0.25（负 mip bias 提锐度）

**前置依赖**：REQ-A

**验收**：

- 静态场景：边缘 aliasing 显著降低，无 ghosting
- 慢速 camera 移动：history reproject 正确，无明显 trail
- 快速 camera 移动：会有 ghosting（这版预期问题，REQ-C 解决）

**风险**：低。这是教科书最简版，无 corner case。

**可独立合 PR**：是。早做能立刻获得 visual baseline。

### REQ-C · TAA 5 大改进（防 ghosting + 防 blurry）

**目标**：把 REQ-B 的 baseline 升级到工业级水准，覆盖 [04 §4.5-4.10](04-TAA.md#45-改进-1reprojection-closest-depth) 的 5 个改进点。

**改动**：

- **改进 1**：closest-depth 3×3 search（取 closest depth 像素的 motion vector）
- **改进 2**：history sampling 切到 Catmull-Rom 5-tap bicubic
- **改进 3**：scene sampling 加 3×3 邻域 + 计算 m1/m2 moments
- **改进 4**：variance clip + clamp（mean ± gamma×sigma + neighborhood AABB clamp）
- **改进 5**：动态 resolve weight（temporal filter + inverse luminance + luminance difference）
- 新增 sharpen post-process pass（luminance-based 高通）
- 实现 unjitter texture UVs 工具（dFdx/dFdy 抵消 jitter）

**前置依赖**：REQ-B

**验收**：

- 快速 camera 移动：ghosting 显著减少
- 细薄结构（1-pixel 线条）：jitter flicker 受抑制
- HDR 极亮 pixel：firefly 累积放大被 inverse luminance 抑制
- 整体画面比 REQ-B 锐度提升

**风险**：中。每个改进单点风险低，但 5 个组合调参需要时间。

**可独立合 PR**：是，可拆成 5 个 sub-PR 逐项合并。

### REQ-D · Volumetric Fog（完整 5 步 pipeline）

**目标**：实现 [03](03-volumetric-fog-pipeline.md) 的完整 volumetric fog pipeline。

**改动**：

- 新增 4 张 3D texture（froxel_data / light_scattering / history_light_scattering / integrated_volumetric_fog，~32 MB）
- 烘焙 3D Perlin noise texture（~4 MB，启动期一次性 compute）
- 新增 5 个 compute pass：
  1. Data injection（常量雾 + 高度雾 + 体积雾 box + Perlin 扰动）
  2. Light scattering（复用 clustered lighting bins/tiles + Henyey-Greenstein phase + cubemap shadow）
  3. Spatial filter（X/Y 5×5 高斯模糊）
  4. Temporal filter（跨帧 reproject + blend_factor=0.05）
  5. Light integration（pre-integrated ray marching + Beer-Lambert）
- Apply to scene：lighting shader 末端加 `color × fog.a + fog.rgb`
- Frustum → froxel UVW 工具（exponential Z 切分）
- 雾参数 UI / Scene 字段（density / height_falloff / phase_g / box list）

**前置依赖**：

- REQ-A（temporal infrastructure，特别是 history texture）
- frame-graph REQ-038（barrier 自动推导）— 5 个 compute pass + lighting apply 的 DAG 必须靠 frame graph 推
- gpu-driven-rendering REQ-F（clustered lighting）— scattering 复用 cluster 数据
- shadows REQ-F（cubemap shadow）— fog shadow 受体

**验收**：

- 室外场景：高度雾 + Perlin 飘动效果自然
- 室内场景：光从窗户/门射入，god rays 清晰
- 配合 cubemap shadow：动态光源移动时雾的阴影正确跟随
- 性能：1080p ~1.5ms（参考 Wronski 2014 论文）

**风险**：高。5 个 compute pass + 跨帧 history + 跟两个上游调研协同 — *这是本路线最复杂的 REQ*。

**可独立合 PR**：建议拆 3 个 sub-PR：
- D.1：3D texture + Perlin 烘焙 + Step 1 injection
- D.2：Step 2 scattering + Step 3 spatial filter
- D.3：Step 4 temporal + Step 5 integration + scene apply

### REQ-E · TAA + Volumetric Fog 协同优化

**目标**：把 TAA 和 volumetric fog 的 *temporal accumulation* 协同起来，让两者互相加成。

**改动**：

- Volumetric fog 引入更激进的 dithering（blue noise 抖动 + 时间动画化）
- TAA 跨帧累积自动平掉 fog dithering 噪声
- 实现 history 失效检查的 *统一策略*（disocclusion + camera cut + out-of-frustum）
- 共享 blue noise 资产（避免重复加载）
- Profile 验证：dithering 加强后的画质 / 噪声 trade-off

**前置依赖**：REQ-C + REQ-D

**验收**：

- Volumetric fog 在静态画面下无可见噪声 / banding
- TAA 不被 fog 的 high-frequency dithering 干扰
- 两者协同后，整体画面 smoother 且更细腻

**风险**：低。是已有机制的 fine-tuning。

**可独立合 PR**：是。

### REQ-F · Disocclusion / 半透明 / specular 单独路径（工业级 corner case）

**目标**：处理 TAA 的剩余 corner case，达到接近 UE / Frostbite 的画质。

**改动**：

- Motion vector 突变检测 → disocclusion 区域 history weight 归零
- 半透明物体跳过 TAA / 单独 pass
- Specular highlight 单独 temporal filter（避免高光跨帧位置变）
- HDR firefly tonemap-based weighting（已经在 REQ-C 改进 5 里有，这里加强）
- TAA reset 机制（camera cut 检测）

**前置依赖**：REQ-C（TAA 改进版基础）

**验收**：

- 角色快速从遮挡物后走出：disocclusion 区域无 trail
- 半透明粒子 / 玻璃：无 ghost
- 高光：跨帧抖动减弱
- 整体跟商业引擎对比，剩余痕迹只在极端 corner case

**风险**：中。每个 corner case 单独处理简单，但全部覆盖需要时间。

**可独立合 PR**：是，可拆 4 个 sub-PR。

## 6.3 依赖图

```
multi-threading/08（IGpuTimeline + RetirePoint）
       │
frame-graph REQ-035（FramePass）
       │
       ↓
   ┌───────────────────────────────────────┐
   │ REQ-A · Temporal 共享基础设施          │
   │ jitter / motion vec / history /        │
   │ blue noise / golden ratio              │
   └────────┬──────────────────────────────┘
            │
            ├──────────────────────────────┐
            ↓                              ↓
       REQ-B · TAA baseline           （等 frame-graph REQ-038
            │                          + clustered lighting
            ↓                          + cubemap shadow）
       REQ-C · TAA 5 大改进                ↓
            │                         REQ-D · Volumetric Fog
            ↓                              │
       REQ-F · TAA corner case             │
            │                              │
            └──────────┬───────────────────┘
                       ↓
              REQ-E · TAA + Fog 协同
```

**关键观察**：

- REQ-A 是 *唯一硬前置*，其余可按收益穿插
- REQ-B/C 跟 REQ-D 完全并行（不互相阻塞）— TAA 早做能先拿 visual baseline
- REQ-D 的前置依赖 *最重*（frame-graph + clustered + shadow 三个上游）
- REQ-E 是 *最后的 polish*，必须 C + D 都就位

## 6.4 何时触发 REQ-A 立项

按 [`README §"何时触发本调研立项"`](README.md#何时触发本调研立项) 的判断，触发条件：

- **强触发**：[Phase 1](../../main-roadmap/phase-1-rendering-depth.md) 主路径（PBR / shadow / clustered lighting）完成 → 立 REQ-A + REQ-B
- **强触发**：测出 fragment temporal aliasing（树叶闪烁、铁丝网鬼影）影响画质 → 立 REQ-A + REQ-B + REQ-C
- **强触发**：引入室内场景 / 大空间 → 立 REQ-A + REQ-D（god rays 是室内必需）
- **中触发**：视觉品质升级阶段，需要 AAA 级表现
- **中触发**：引入 SSR / SSAO / DoF — 这些都依赖 temporal infrastructure，REQ-A 一次铺好多个特性受益
- **弱触发**：纯学习 / 探索目的

任一条强触发条件命中，REQ-A 应紧随其后立项。当前 Phase 1 路线图（PBR / shadow / IBL / bloom）**没有 temporal REQ**，需要新增条目。

## 6.6 工业级延伸（不在本路线范围）

完成 REQ-F 后离 *最现代 temporal 子系统* 还有以下方向，建议不进 v1 路线：

- **TAAU**（Temporal AA Upsampling）：渲染分辨率低于显示分辨率，TAA 同时做 AA + 上采样
- **DLSS / FSR / XeSS 集成**：AI / 频域上采样
- **SSR 共享 temporal 基础设施**：screen-space reflection 跨帧累积
- **SSAO 跨帧累积**：noisy AO 跨帧平均
- **DoF 跨帧累积**：bokeh 跨帧累加样本
- **Stochastic skin / hair shadow**：dithering + TAA 平掉
- **Path-traced GI 跨帧累积**：实时 GI 的 temporal denoising
- **Volumetric cloud + 大气**：雾的延伸，跟 Hillaire 大气模型协同

这些都是 *优化层 / 高级特性*，等 v1 跑稳定且有真实需求再考虑。

## 6.7 风险与不确定性

### 风险 1：Volumetric Fog 的上游依赖最重

REQ-D 同时依赖 **frame-graph REQ-038 + clustered lighting REQ-F + cubemap shadow REQ-F**。三个上游任一拖延都会阻塞 REQ-D。

**缓解**：

- 优先推 frame-graph + clustered + shadow（已经在各自调研里规划）
- REQ-A/B/C 可以在等待期间独立推进，不阻塞

### 风险 2：TAA 调参陷阱

TAA 的 5 大改进有大量 *魔法数字*：

- `gamma = 1`（variance clip 强度）
- `0.25 / 0.85`（temporal filter weight 范围）
- `0.9 / 0.1`（基础 history blend）
- `mipLodBias = -0.25`（sharpness vs aliasing 平衡）

不同场景 / 内容下最优值差异大。需要：

- 配置化 + 调试 UI（可视化每个 buffer 中间态）
- 对比测试套件（标准场景 + reference frame）
- Profiling 工具（Renderdoc / Nsight 抓帧）

### 风险 3：跨硬件画质 / 性能差异

- Volumetric fog 的 5 个 compute pass 对 *bandwidth* 敏感，集显 / 移动 GPU 性能差异大
- TAA 的 16-bit 中间精度在某些硬件可能不够（需要 fallback 到 32-bit）
- Catmull-Rom 5-tap 在低端 GPU 可能太重（fallback 到 bilinear）

**缓解**：

- 引入质量档位（high / medium / low），不同档位选不同算法
- 启动期 hardware feature detection
- Profile 在 NVIDIA + AMD + Intel 三套硬件

### 风险 4：调试难度

Temporal bug 的症状 *跟单帧 bug 完全不同*：

- 偶发闪烁（reproject 边界 case）
- 慢速 ghost（history weight 不对）
- 画面慢慢偏色（accumulation 数值漂移）

需要：

- per-frame buffer dump 机制（抓某一帧的 history / motion vector / scene color）
- 跟 reference image 的对比工具
- 单帧重放（保存某帧的全部 input，离线复现）

## 6.8 一句话总结

**Temporal techniques = 跨帧累积样本换画质。LX 当前 Level 0（无 temporal）；推到 Level 4（Variance clip + 5 大改进 + Wronski volumetric fog）需要 6 个 REQ + 三个上游调研协同。基础设施 REQ-A 是唯一硬前置，可早做；REQ-B/C 早做能先拿 TAA baseline；REQ-D 等 frame-graph + clustered + shadow 全就位。本调研是 *消费方*，不引入新架构概念，但把前面 5 个调研组合成现代视觉特性。**
