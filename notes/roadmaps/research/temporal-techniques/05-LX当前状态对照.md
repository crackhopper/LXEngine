# 05 · LX 当前状态对照

> 阅读前提：[01](01-temporal-范式.md) ~ [04](04-TAA.md) 讲完了 temporal 范式 + volumetric fog + TAA。本文综合盘点 LX gap。

## 5.1 LX 现状速查

LX 当前 *零 temporal 能力*：

- 单 forward pass，每帧完全独立渲染
- 无 jitter / motion vector / history texture
- 无 volumetric fog（雾固定颜色叠加都没有）
- 无 TAA / 任何 anti-aliasing 路径
- 无 blue noise / Perlin noise 资产
- Phase 1 路线图里 *甚至没有专门的 temporal REQ*

## 5.2 综合 gap 总表

### Temporal 共享基础设施

| 维度 | LX 现状 | 需要 | gap |
|------|---------|------|-----|
| Camera jitter API | 无 | projection matrix 修改 + Halton sequence | 中 |
| Previous-frame view-projection 缓存 | 无 | per-frame 双 buffer | 小 |
| Motion vector buffer (R16G16) | 无 | 新 RT 类型 | 中 |
| Velocity compute pass (camera motion) | 无 | reconstruct from depth + view-proj | 中 |
| Per-vertex velocity 输出 | 无 | vertex/mesh shader 扩展 | 中 |
| History texture 双 buffer | 无 | 跨帧资源 + RetirePoint | 中 |
| Blue noise 资产 + 采样 | 无 | 2D texture + 工具函数 | 小 |
| Golden ratio 时间偏移 | 无 | shader util | 小 |

### Volumetric Fog

| 维度 | LX 现状 | 需要 | gap |
|------|---------|------|-----|
| 3D texture 创建 + 采样 | 无 | `IGpuResource` + 3D image view | 中 |
| Froxel 数据结构 | 无 | 4 张 3D texture（~36 MB） | 中 |
| Frustum → froxel UVW 工具 | 无 | shader math（exponential Z） | 小 |
| Phase function (Henyey-Greenstein) | 无 | shader util | 小 |
| Light scattering compute pass | 无 | 复用 clustered + phase + shadow | 中 |
| Spatial Gaussian filter compute | 无 | 5×5 模糊 | 小 |
| Temporal filter compute (跨帧 reproject) | 无 | 历史 froxel reproject | 中 |
| Pre-integrated ray marching compute | 无 | Z 累加 Beer-Lambert | 中 |
| Apply to scene shader | 无 | lighting shader 末端调用 | 小 |
| Perlin 3D 烘焙 | 无 | one-shot compute shader | 小 |

### TAA

| 维度 | LX 现状 | 需要 | gap |
|------|---------|------|-----|
| TAA shader 链 | 无 | 6 步 compute shader | 中 |
| Closest-depth 3×3 search | 无 | shader util | 小 |
| Catmull-Rom history filter | 无 | shader util | 小 |
| Variance clip 工具 | 无 | shader util | 小 |
| Resolve 多 filter（temporal/luminance） | 无 | shader util | 小 |
| Sharpen post-process | 无 | post-process pass | 小 |
| Negative mip bias 配置 | 无 | sampler 配置 | 极小 |

总计 gap 不算大（大部分是 *组合性* 工作），但 *依赖前面调研* 全部就位才能完整跑通。

## 5.3 LX 已经做对的、能直接对接的部分

| LX 已有 | 在本调研里的作用 |
|---------|----------------|
| `IGpuResource` 统一契约 | 3D texture / motion vector buffer / history buffer 都走同接口 |
| `IGpuTimeline`（multi-threading/08） | history texture 跨帧 retire 跟踪 |
| `RetirePoint` | 双 buffer 切换的生命周期管理 |
| Clustered lighting（gpu-driven REQ-F） | volumetric fog 的 light scattering 直接复用 |
| Cubemap shadow（shadows REQ-F） | volumetric fog 的 shadow 采样直接复用 |
| Material system + reflection | shader 反射框架共用 |
| `Scene` 容器 | 雾参数 + camera 上传源 |

## 5.4 跟现有 LX REQ 的对照

LX Phase 1 路线图里跟本调研 *直接对应* 的 REQ 几乎没有：

| 现有 REQ | 跟本调研关系 |
|---------|------------|
| 无 | volumetric fog 完全未在 Phase 1 路线图，需新立 REQ |
| 无 | TAA 完全未在 Phase 1 路线图，需新立 REQ |
| [REQ-118 PBR 完整管线](../../main-roadmap/phase-1-rendering-depth.md#req-118--pbr-完整管线) | TAA blur 影响 PBR 高光，需协同（多 channel weight） |
| [REQ-119 G-Buffer / 延迟渲染](../../main-roadmap/phase-1-rendering-depth.md#req-119--g-buffer--延迟渲染路径) | TAA 在 deferred 后端最常用 |

**重要观察**：本调研引入的能力 *Phase 1 路线图本来没有计划*，需要在路线图里新增条目。

## 5.5 跟前面 5 个调研的耦合

### 跟 multi-threading/08

| 耦合点 | 强度 |
|------|------|
| History texture 跨帧 retire | **强** |
| Motion vector buffer 跨帧 | 中 |
| Volumetric fog history 跨帧 | 中 |

### 跟 frame-graph 调研

| 耦合点 | 强度 |
|------|------|
| Volumetric fog 5 个 compute pass + lighting apply = *6 个 pass DAG* | **强** |
| TAA 是 lighting 之后的 post-process pass | 中 |
| 跨帧资源（history）frame graph 表达 | **强** |
| 自动 barrier 推导（REQ-038）必须先就位 | **强** |

### 跟 async-compute 调研

| 耦合点 | 强度 |
|------|------|
| 所有 temporal compute pass 是 async candidate | 中 |
| Compute pipeline 路径直接复用 | **强** |

### 跟 gpu-driven-rendering 调研

| 耦合点 | 强度 |
|------|------|
| Volumetric fog scattering 复用 clustered lighting bins/tiles | **极强** |
| Phase function 跟 BRDF 路径协同 | 弱 |

### 跟 shadows 调研

| 耦合点 | 强度 |
|------|------|
| Volumetric fog 采样 cubemap shadow → god rays | **强** |
| Shadow pass 不用 TAA（depth-only） | 无 |

## 5.6 LX 当前位置（按能力层级）

```
Level 0: 无 temporal                                ← LX 在这里
Level 1: 基础 temporal infrastructure（jitter / motion vector / history buffer）
Level 2: Volumetric fog（最简版）                   ← Wronski 2014 论文水平
Level 3: + TAA（最简版）
Level 4: + Variance clip / 5 大改进                  ← Ch10 + Ch11 教科书目标
Level 5: + Sharpen / disocclusion 处理 / specular 单独路径
Level 6: + 跟 SSR / SSAO / DoF 共享 temporal 基础设施
```

要从 0 推到 4 需要本调研全部 6 个候选 REQ。

## 5.7 一句话定位

**本调研是 *消费方*，依赖前面 5 个调研全部就位才能完整跑通**：

- frame-graph 提供多 pass + 跨帧资源
- multi-threading/08 提供 history retire 模型
- async-compute 提供 compute pipeline
- gpu-driven-rendering 提供 clustered lighting 数据
- shadows 提供 cubemap shadow

但 *基础 TAA*（jitter + motion vector + 简单 history）可以早做一些，作为 visual quality baseline。
