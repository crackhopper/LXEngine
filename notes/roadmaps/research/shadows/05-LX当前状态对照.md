# 05 · LX 当前状态对照

> 阅读前提：[01](01-shadow技术全景.md) ~ [04](04-sparse-resources.md) 已经讲完 shadow + sparse 全部技术细节。本文综合盘点 LX gap。

## 5.1 LX 当前阴影现状

**完全空白**：

- 无任何形式的 shadow rendering
- 无 cubemap textures（即便 IBL [REQ-105](../../main-roadmap/phase-1-rendering-depth.md#req-105--environment-map-loader--scene-level-ibl-资源) 也只是预留）
- 无 layered rendering / multiview
- 无 sparse resources / sparse residency
- Phase 1 路线图里 shadow 相关的所有 REQ（103/104/109）当前 *都没立项*

## 5.2 gap 总表

### Shadow rendering 维度

| 维度 | LX 现状 | 本调研需要 | gap |
|------|---------|----------|-----|
| Shadow map（2D） | 无 | 入门级 | 大 |
| Shadow map（cubemap） | 无 | point light 需要 | 大 |
| Cubemap array | 无 | 多光源 | 大 |
| Layered rendering（`gl_Layer`） | 无 | 多 layer 一次性写 | 中 |
| Multiview rendering | 无 | 6 face vertex 共享（可选） | 中 |
| Depth-only pipeline | 无（forward 走 vertex+fragment） | shadow pass 必须 | 中 |
| Depth bias / slope-scale bias | 无 | 防 shadow acne | 小 |
| Shadow filtering (PCF) | 无 | 软化锯齿 | 中 |
| CSM（cascaded shadow maps） | 无 | directional light 大场景 | 中 |
| Vector → depth 工具函数 | 无 | cubemap shadow 必须 | 小 |
| Cubemap face mask 工具 | 无 | cubemap face culling | 小 |

### Sparse resources 维度

| 维度 | LX 现状 | 本调研需要 | gap |
|------|---------|----------|-----|
| Sparse-capable resource flag | 无 | `SPARSE_RESIDENCY_BIT` | 中 |
| Page pool 管理 | 无 | VMA 批量分配 + 索引 | 中 |
| `vkQueueBindSparse` | 无 | bind 命令封装 | 中 |
| Bind semaphore | 无 | 跟 timeline 协同 | 中 |
| Sparse queue 选择 | 无 | sparse-binding queue family | 中 |
| Per-light importance metric | 无 | cluster-based 算法 | 中 |
| 跨帧 sparse 资源 retire | 无 | RetirePoint 扩展 | 中 |

整体距离：**比 GPU-driven 调研更下游**，依赖 GPU-driven + clustered 已经就位。

## 5.3 LX 已经做对的、能直接对接的部分

| LX 已有 | 在本调研路线里的作用 |
|---------|--------------------|
| `IGpuResource` 统一契约 | sparse-bound cubemap 也走同接口（加 sparse flag） |
| `IGpuTimeline`（multi-threading/08）| bind_semaphore 跟 graphics submit 协同 |
| `RetirePoint`（multi-threading/08） | sparse pages 也需要按 retire 跟踪 |
| Material system | shadow shader 跟 PBR shader 共用反射框架 |

| Cluster framework（[clustered lighting REQ-F](../gpu-driven-rendering/06-演进路径.md#req-f--clustered-lighting)） | importance metric 直接复用 cluster 计算 |
| Mesh shader / compute culling pipeline（[gpu-driven REQ-D / REQ-G](../gpu-driven-rendering/06-演进路径.md)） | shadow 走同 pipeline |

| REQ | 内容 | 跟本调研关系 |

**重要观察**：REQ-103 / REQ-104 / REQ-109 当前 *实现细节都没定*。本调研演进路径就是给它们提供具体实施方案。

## 5.5 跟其它四个调研的耦合

### 跟 multi-threading/08

| 耦合点 | 强度 |
|------|------|
| Sparse bind_semaphore 跟 graphics submit 走 timeline | **强** |
| Sparse pages 需要按 RetirePoint 跟踪生命周期 | 中 |
| 跨帧 cubemap 复用（上一帧 → 本帧） | 中 |

### 跟 frame-graph 调研

| 耦合点 | 强度 |
|------|------|
| Shadow rendering 是 *新的 pass type*，需要 frame graph 表达 | **强** |
| 每光源 6 个 indirect draw → 6 × N 个 draw call，frame graph 调度 | 强 |
| Cubemap array 是跨帧资源（本帧画 → 本帧读） | 中 |
| 自动 barrier 推导（REQ-038）必须先就位 | **强** |

### 跟 async-compute 调研

| 耦合点 | 强度 |
|------|------|
| Per-light culling compute pass 是 async candidate | 中 |
| Sparse bind 可走 sparse queue（如果硬件支持），跟 graphics 并行 | 中 |
| Compute pipeline 路径直接复用 async-compute REQ-A | **强** |

### 跟 gpu-driven-rendering 调研

| 耦合点 | 强度 |
|------|------|
| Mesh shader pipeline 复用（[gpu-driven REQ-G](../gpu-driven-rendering/06-演进路径.md#req-g--mesh-shader-pipeline--实战)） | **极强** |
| Compute culling 路径复用（[gpu-driven REQ-D](../gpu-driven-rendering/06-演进路径.md#req-d--compute-culling-基础frustum--back-face)） | **极强** |
| Meshlet 资产复用（[gpu-driven REQ-A](../gpu-driven-rendering/06-演进路径.md#req-a--meshlet-资产-pipeline)） | **极强** |
| Indirect drawing 复用（[gpu-driven REQ-B](../gpu-driven-rendering/06-演进路径.md#req-b--indirect-draw--storage-buffer-基础)） | **极强** |
| Cubemap face culling 工具是新增 | — |

**90% 的 shadow 渲染基础设施 = gpu-driven 已设计的能力**。本调研的真正"新增"是 cubemap shadow 数据结构、layered rendering、sparse residency 这三块。

## 5.6 LX 当前位置（按能力层级）

```
Level 0: 无 shadow                             ← LX 在这里
Level 1: 单 directional shadow map（hard）
Level 2: + PCF / cascade
Level 3: + point light cubemap shadow（少光源）
Level 4: + 多光源 cubemap array（固定分辨率）
Level 5: + sparse residency + per-light dynamic resolution    ← Ch8 教科书目标
Level 6: + raytraced shadow（fallback / 高保真）
```

要从 0 推到 5 需要本调研全部 7 个候选 REQ。

## 5.7 一句话定位

**本调研是 *最下游消费方***：依赖前面四个调研全部就位才能完整实施。但 *基础 shadow*（候选 REQ-A + B）可以早做（不需要 mesh shader / sparse），先打通 directional shadow map 路径。
