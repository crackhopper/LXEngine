# 实现技巧（Impl Tricks）

> 这一组文档从 `LXEngine` 现有代码里抽取出一些“当时看起来很朴素、但其实在工程上很有味道”的小手法。
> 关注点不是某个大子系统的设计，而是 **具体写法、取舍、权衡**——可以当成一本薄薄的内部菜谱。

每一篇通常包含四个部分：

1. **现象 / 场景**：这段代码长什么样、在哪个位置出现。
2. **在解决什么问题**：没有这个手法的话会变成什么样。
3. **手法本身**：用一两个术语概括出来，方便以后在别的项目里认出它。
4. **权衡 / 退化路径**：什么时候不要用、什么时候会退化成反模式。
